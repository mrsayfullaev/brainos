# Анализ готовности модулей BrainOS — реальная картина

> Глубокий анализ кодовой базы. Дата: март 2025.

---

## ⚠️ Критическое: почему «всё готово, но ничего не работает»

Документ отражает **реальное состояние** проекта после детального разбора кода. Многие модули выглядят готовыми, но на практике работают частично или создают некорректные данные.

---

## 1. Архитектурная проблема: action не передаётся в handlers

### Суть

Роутер (`router.ts`) анализирует намерение и получает:
- `module` — task, wallet, note и т.д.
- `action` — create, list, read, update, delete
- `confidence` — 0–100

**Но `action` никуда не передаётся.** Handlers вызываются так:

```ts
const handler = MODULE_HANDLERS[intent.module];
await handler(ctx, user, message);  // action не передаётся!
```

### Следствия

| Запрос пользователя | Ожидание | Фактическое поведение |
|--------------------|----------|------------------------|
| «покажи задачи» | Список задач | Создаётся задача с заголовком «покажи задачи» |
| «покажи заметки» | Список заметок | Создаётся заметка с текстом «покажи заметки» |
| «мои напоминания» | Список напоминаний | Пытается создать напоминание, возможны ошибки |
| «покажи расходы» | Статистика | Ошибка (amount=0) → «There was an error...» |

### Handlers для list/read есть, но не используются

| Модуль | Существующий handler | Регистрация |
|--------|----------------------|-------------|
| task | `handleShowTasks` | ❌ Не зарегистрирован |
| wallet | `handleShowExpenses` | ❌ Не зарегистрирован |
| remind | `handleShowReminders` | ❌ Не зарегистрирован |

В роутере у каждого модуля только один handler — всегда «create».

## 2. Что реально работает

### 2.1 Создание (create) — работает

| Модуль | Пример | Результат |
|--------|--------|-----------|
| task | «задача купить молоко» | ✅ Создаётся задача |
| note | «заметка идея для проекта #важное» | ✅ Создаётся заметка |
| wallet | «кофе 290», «+80000 зарплата» | ✅ Создаётся транзакция |
| remind | «напомни завтра в 10:00 встреча» | ✅ Создаётся напоминание |

### 2.2 Изменение статуса задачи — работает

Фразы «готово», «выполнено», «в процессе», «отмени» обрабатываются в `handleTaskMessage` и меняют статус задачи.

### 2.3 Стандартный AI-ответ — работает

Если intent не определён (confidence < 70) или модуль недоступен по тарифу — вызывается `handleStandardMessage`: обычный ответ AI без модулей.

## 3. Что не работает или работает неправильно

### 3.1 List/Read — создаётся мусор или ошибка

| Модуль | Запрос | Результат |
|--------|--------|-----------|
| task | «покажи задачи», «мои задачи» | Создаётся задача-мусор |
| note | «покажи заметки», «список заметок» | Создаётся заметка-мусор |
| wallet | «покажи расходы», «статистика» | Ошибка (amount=0) |
| remind | «мои напоминания» | Попытка создать напоминание, возможны ошибки |

### 3.2 Зависимости от внешних сервисов

| Компонент | Требование | Риск |
|-----------|------------|------|
| **config.ts** | `GIGACHAT_CLIENT_ID`, `GIGACHAT_CLIENT_SECRET` — обязательны | Без них приложение не стартует |
| GigaChat | Доступ из РФ | Заблокирован за пределами РФ → fallback на OpenAI |
| OpenAI | `OPENAI_API_KEY` | Без него AI не работает |
| Intent + парсеры | systemLLMGenerate (GigaChat → OpenAI) | При сбое обоих — ошибка |

### 3.3 Wallet cron — заглушки

- `sendWeeklyDigests` — текст «Здесь будет сводка...», без вызова `generateWeeklyDigest`
- `checkBudgetAlerts` — закомментирован, не выполняется
- `sendMonthlyForecast` — текст «Здесь будет прогноз...», без вызова `forecastMonthlyExpenses`

### 3.4 Research Agent

- `webSearch()` → `[]`
- `webFetchContent()` → `''`
- Команда `/research` отрабатывает, но поиск не выполняется.

### 3.5 Workflow

- `create_task` — заглушка, задача не создаётся
- `save_to` — заглушка, сохранение не выполняется

## 4. Обязательные требования для запуска

### 4.1 Переменные окружения (config.ts)

| Переменная | Обязательность |
|------------|----------------|
| `DATABASE_URL` | ✅ Обязательно |
| `TELEGRAM_BOT_TOKEN` | ✅ Обязательно |
| `OPENAI_API_KEY` | ✅ Обязательно |
| `GIGACHAT_CLIENT_ID` | ✅ Обязательно |
| `GIGACHAT_CLIENT_SECRET` | ✅ Обязательно |
| `ENCRYPTION_KEY` | ✅ Обязательно (32+ символа hex) |

Без любой из них приложение не запустится.

### 4.2 Онбординг и согласие

1. **Согласие на ПД** — без нажатия «Согласен» бот блокирует все действия.
2. **Онбординг** — имя, язык, тон, длина, эмодзи, структура, стиль, детализация, бонусный модуль.
3. **Сессия** — при потере сессии (перезапуск, долгая пауза) пользователь может «застрять» в онбординге.

### 4.3 Тариф Free

- 5 модулей: task, note, wallet, health, contact
- 1 бонусный модуль на выбор
- 100 AI-запросов в месяц

## 6. Рекомендации по исправлению

### Критический приоритет

1. **Передавать action в handlers**
   - Расширить сигнатуру: `handler(ctx, user, message, intent)`
   - В handlers ветвить логику по `intent.action`

2. **Подключить list/read handlers**
   - В task: при `action === 'list'` вызывать `handleShowTasks`
   - В wallet: при `action === 'list'` вызывать `handleShowExpenses`
   - В remind: при `action === 'list'` вызывать `handleShowReminders`

3. **Защита от мусорных записей**
   - В note: не создавать заметку при явных list-запросах («покажи», «список», «мои»)
   - В task: аналогичная проверка перед созданием

### Высокий приоритет

4. **Wallet cron** — вызывать реальные функции вместо заглушек
5. **Research** — реализовать `webSearch` и `webFetchContent`
6. **Workflow** — реализовать `create_task` и `save_to`

### Средний приоритет

7. **GigaChat** — сделать опциональным (fallback только на OpenAI)
8. **Scheduler** — добавить cron для Sub, Contact, Vocab, News
